# 图片加载失败问题诊断与解决方案

## 问题描述

在移动应用的"基础信息·录入/上传"页面中，附件上传部分出现"图片加载失败"的错误提示。

错误日志：
```
index.vue:1126 图片加载失败: {type: "error", timeStamp: 7317, target: {…}, currentTarget: {…}, mark: {…}, …}
```

## 问题分析

### 1. 系统配置

**后端配置** (`ruoyi-admin/src/main/resources/application.yml`)：
- Profile 路径: `E:\HLDX2.0\Technical Data Share`
- 上传路径: `E:\HLDX2.0\Technical Data Share\upload`
- 静态资源访问: `/profile/**` 映射到 Profile 路径
- 上传接口返回格式:
  - `url`: 完整URL（如 `http://192.168.1.193:8080/profile/upload/2026/02/27/xxx.jpg`）
  - `fileName`: 相对路径（如 `/profile/upload/2026/02/27/xxx.jpg`）

**前端配置** (`ruoyi-app/config/index.js`)：
- 开发环境 baseUrl: `http://192.168.1.193:8080`
- 生产环境 baseUrl: `https://your-domain.com/prod-api`

### 2. 可能的原因

#### 原因1：URL 拼接不一致
- 后端返回的 `url` 字段包含完整的服务器地址
- 如果后端的 `serverConfig.getUrl()` 返回的地址与前端配置的 `baseUrl` 不一致，会导致图片路径错误

#### 原因2：文件路径不存在
- 上传的文件可能保存失败
- 文件路径在数据库中保存错误

#### 原因3：网络问题
- 服务器地址 `192.168.1.193:8080` 不可访问
- 小程序网络权限限制

#### 原因4：静态资源配置问题
- 后端的静态资源映射配置不正确
- 文件权限问题导致无法访问

## 已实施的修复方案

### 1. 优化 URL 拼接逻辑

**修改文件**: `ruoyi-app/pages/trialProcess/index.vue`

**修改点1**: 上传文件时优先使用 `fileName`（相对路径）
```javascript
// 第1004-1023行
uploadFile(filePath).then(data => {
  // 优先使用 fileName（相对路径），由前端统一拼接完整URL
  // 这样可以避免后端返回的 url 地址不正确的问题
  let fileUrl = data.fileName || data.url || filePath
  
  // 如果是相对路径，拼接 baseUrl
  if (fileUrl && !fileUrl.startsWith('http://') && !fileUrl.startsWith('https://')) {
    fileUrl = this.baseUrl + fileUrl
  }
  
  // 调试日志：打印上传结果
  console.log('文件上传成功:', {
    fileName: data.fileName,
    url: data.url,
    finalUrl: fileUrl
  })
  
  const newFile = {
    name: fileName,
    url: fileUrl,
    time: new Date().toISOString()
  }
  // ...
})
```

### 2. 增强 getFileUrl 方法

**修改点2**: 添加详细的调试日志
```javascript
// 第1072-1092行
getFileUrl(url) {
  if (!url) {
    console.warn('getFileUrl: URL为空')
    return ''
  }
  
  // 如果已经是完整URL，直接返回
  if (url.startsWith('http://') || url.startsWith('https://')) {
    return url
  }
  
  // 如果是相对路径，拼接baseUrl
  const fullUrl = this.baseUrl + url
  console.log('getFileUrl 拼接:', { 
    原始URL: url, 
    baseUrl: this.baseUrl, 
    完整URL: fullUrl 
  })
  return fullUrl
}
```

## 常见问题

### Q1: 为什么要优先使用 fileName 而不是 url？

A: 因为后端的 `serverConfig.getUrl()` 可能配置不正确或返回内网地址，而前端的 `baseUrl` 是我们明确配置的正确地址。使用相对路径 + 前端拼接可以保证 URL 的正确性。

### Q2: 如何临时禁用图片懒加载进行测试？

A: 在 `<image>` 标签中移除 `lazy-load` 属性：
```vue
<image
  :src="getFileUrl(file.url)"
  class="file-thumb-img"
  mode="aspectFill"
  @error="onImageError"
  @load="onImageLoad"
/>
```

### Q3: 如何在生产环境使用 HTTPS？

A: 
1. 修改 `ruoyi-app/config/index.js`，设置 `IS_PRODUCTION = true`
2. 配置正确的生产环境域名 `baseUrl: 'https://your-domain.com/prod-api'`
3. 确保后端也部署在该域名下，并配置了 HTTPS 证书
4. 在微信公众平台配置合法域名
