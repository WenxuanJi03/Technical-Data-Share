# 轮毂识别深度优化方案 (第二版)

## 问题现状

第一次优化后,识别率仍然很低:
- 最高相似度: **5.6%**
- 其他候选: 4.0%, 3.8%
- 状态: **完全无法识别** ❌

## 深度优化策略

### 1. 极限增加特征点
```java
ORB_FEATURES: 800 → 1500 → 2500  // 提升212%!
```
- 轮毂有大量复杂纹理
- 更多特征点 = 更robust的匹配
- 即使部分特征丢失也能匹配

### 2. 极限放宽匹配参数
```java
MATCH_RATIO_THRESHOLD: 0.7 → 0.75 → 0.80  // 极限放宽
MIN_GOOD_MATCHES: 10 → 8 → 6  // 降到最低
MIN_SCORE_THRESHOLD: 0 → 0.015 → 0.01  // 1%即返回
```

### 3. 激进图像预处理

#### 新增处理步骤:

**步骤1: 尺寸标准化**
```java
// Resize到800px以内,统一处理尺寸
```
- 消除尺寸差异
- 提升性能

**步骤2: 双重直方图均衡化**
```java
equalizeHist → equalizeHist (两次!)
```
- 更激进地处理光照
- 消除反光影响

**步骤3: 自适应阈值二值化**
```java
adaptiveThreshold + 混合原图
```
- 增强边缘对比
- 突出轮毂纹理

**步骤4: 强锐化**
```java
// 锐化核从 [-0.5, 5, -0.5] → [-1, 9, -1]
```
- 更激进的边缘增强
- 更明显的特征点

### 4. 激进评分算法

```java
// 优化前
countScore = goodCount / denominator
distanceScore = 1 / (1 + avgDist/50)
rawScore = countScore * distanceScore * 2.0

// 激进优化后
countScore = pow(goodCount / denominator, 0.8)  // 指数降低
distanceScore = 1 / (1 + avgDist/80)  // 极度宽容
rawScore = countScore * distanceScore * 4.0  // 翻倍权重!
```

**效果**:
- 小匹配数也能得到较高分数
- 距离评分极度宽容
- 综合得分翻倍

## 为什么之前识别率这么低?

ORB特征匹配对以下因素非常敏感:
1. **光照变化** - 反光、阴影
2. **角度变化** - 非正面拍摄
3. **尺度变化** - 距离远近
4. **背景干扰** - 工厂环境
5. **纹理复杂度** - 轮毂纹理丰富但重复

本次优化通过:
- ✅ 激进预处理 → 消除光照影响
- ✅ 大量特征点 → 提升鲁棒性
- ✅ 放宽阈值 → 适应现实场景
- ✅ 激进评分 → 更容易匹配成功
